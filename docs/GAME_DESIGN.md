# 3D 캐릭터 기반 랜덤 합성 엘리멘탈 타워디펜스 설계 문서

## 0. 전제와 목표

이 문서는 `3D 캐릭터 기반 랜덤 합성 엘리멘탈 타워디펜스`의 실제 기준 문서로 사용할 수 있도록 작성한 초기 설계안이다. 현재 저장소에는 기존 구현이 없으므로, 본 문서는 `MVP를 안전하게 완성하면서도 시각적 만족감을 잃지 않는 방향`을 기준으로 잡는다.

### 전제

- 플랫폼 1차 목표: `PC 싱글플레이`
- 조작 기준: `키보드 + 마우스`
- 한 판 플레이 타임 목표: `15~25분`
- 카메라 기준: `세미 아이소메트릭 + 캐릭터가 보이는 3D 전장 시점`
- 개발 전략: `MVP를 먼저 완성하고, 그 위에 비주얼과 콘텐츠를 증축`
- 핵심 우선순위: `보는 맛 > 읽히는 전투 > 랜덤 합성 재미 > 콘텐츠 양`

## 1. 게임 한 줄 정의

### 한 줄 정의

플레이어가 3D 전장을 직접 뛰어다니며 랜덤 소환한 엘리멘탈 타워를 배치하고, 같은 타워 3개를 합성해 초월 형태로 진화시키며 웨이브를 막는 `캐릭터 기반 시네마틱 타워디펜스`다.

### 장르 조합 설명

- `3D 액션 조작`
- `타워디펜스`
- `랜덤 드로우/가챠형 빌드 운영`
- `3합성 진화 시스템`
- `짧은 런 기반 전략 게임`

### 차별점 요약

- 플레이어가 화면 밖의 신이 아니라 `전장 안의 관리자 캐릭터`라는 점
- 타워가 단순 스탯 업이 아니라 `실루엣, 파티클, 오라, 공격 패턴`까지 승격되는 점
- 랜덤 소환의 긴장감과 3합성의 쾌감이 `디펜스 구조`에 직접 연결되는 점
- 5단계 타워가 단순 고레벨 유닛이 아니라 `한 판의 서사를 완성하는 연출 피크`가 되는 점

## 2. 핵심 재미(Core Fun) 정의

### 이 게임이 왜 재밌는가

이 게임의 핵심 재미는 `불확실한 드로우를 전장 운영 실력으로 통제해 나가면서, 약한 조각들이 거대한 엘리멘탈 병기로 변하는 성장 체감`에 있다. 플레이어는 매 순간 완벽한 선택지를 받지 못하지만, 제한된 판 위에서 설치 위치, 합성 타이밍, 웨이브 대응, 캐릭터 이동 동선을 통해 결과를 개선한다.

즉, 재미의 구조는 아래 네 가지가 겹쳐서 만들어진다.

1. `랜덤의 기대감`: 다음 소환에서 원하는 재료가 나올지 모른다.
2. `합성의 승격감`: 같은 타워 3개가 하나의 상위 존재로 변하며 전황이 바뀐다.
3. `디펜스의 압박감`: 적이 계속 밀려오므로 판단을 미룰 수 없다.
4. `캐릭터 조작의 현장감`: 플레이어가 직접 뛰어다니며 전장을 관리하므로 몰입감이 높다.

### 반복 플레이 동기

- 같은 규칙이라도 매 판 소환 흐름이 달라 `빌드 결과가 달라진다`.
- 특정 속성 5단계를 만드는 과정 자체가 `런의 목표`가 된다.
- 랜덤이 완전 운빨이 아니라 보정과 운영 실력으로 뒤집을 수 있어 `재도전 의욕`이 생긴다.
- 전투 결과가 숫자만이 아니라 `시각 연출의 누적`으로 보상되므로 성장 만족감이 크다.
- 나중에는 맵, 보스, 속성 해금, 조합 시너지, 난이도 옵션이 `리플레이 동기`를 더한다.

### 요소별 재미 역할

| 요소 | 제공하는 재미 | 설계 포인트 |
|---|---|---|
| 랜덤 소환 | 기대, 긴장, 즉흥 대응 | 완전 랜덤이 아니라 보정이 필요 |
| 3합성 | 수집, 승격, 파워 스파이크 | 합성 순간 연출이 매우 중요 |
| 디펜스 | 압박, 우선순위 판단, 구성 최적화 | 웨이브가 빌드 의사결정을 강요해야 함 |
| 캐릭터 조작 | 현장감, 몰입, 멀티태스킹 | 과도한 액션보다 관리 동선이 핵심 |

## 3. 게임 플레이 루프

### 30초 단위 마이크로 루프

1. 적 처치로 골드를 얻는다.
2. 빈 설치 노드로 이동한다.
3. 랜덤 타워를 소환한다.
4. 합성 가능한 조합이 생기면 합성 여부를 결정한다.
5. 현재 웨이브 성격에 따라 핵심 노드에 화력을 집중하거나 CC를 보강한다.
6. 캐릭터는 다음 설치 지점, 누수 구간, 합성 후보 위치를 오가며 전장을 관리한다.

이 30초 루프는 `소환 -> 확인 -> 재배치 판단 -> 합성 -> 전투 결과 관찰`이 매우 빠르게 반복되어야 한다.

### 1웨이브 단위 루프

1. `준비 구간`
   현재 골드 확인, 빈 노드 확인, 소환/합성 계획 수립
2. `초기 교전`
   적 조합을 보고 현재 세팅이 버티는지 확인
3. `대응 구간`
   누수 구간 근처에 추가 소환, 합성, 핵심 타워 강화
4. `정리 구간`
   남은 적 처리, 생존 여부 확인
5. `웨이브 보상`
   골드/보정 게이지/다음 웨이브 정보 확인

### 1판 전체 메타 루프

1. 시작 로드아웃 또는 해금 속성 확인
2. 초반 기초 빌드 구축
3. 중반부터 특정 속성 축을 정하고 합성 체인을 노린다
4. 후반에 4단계 이상 타워를 중심으로 전장 구조를 재편한다
5. 보스전을 통과하면 런 종료
6. 결과 화면에서 사용 빌드, 합성 최고 단계, 주요 기여 속성 확인
7. 이후 해금, 업적, 신규 속성 또는 난이도로 반복 동기 형성

### 시작 -> 중반 -> 후반 -> 보스전 흐름

| 구간 | 플레이 감정 | 핵심 행동 | 시각 목표 |
|---|---|---|---|
| 시작 | 불안, 기대 | 빠른 설치와 초반 역할 확보 | 작은 정령/토템들이 전장을 채우기 시작 |
| 중반 | 조합 탐색, 운영 집중 | 합성 체인, 역할 분담, 병목 대응 | 속성 색이 전장에 뚜렷이 자리잡음 |
| 후반 | 파워 스파이크, 과부하 | 4단계 이상 중심 재편, 누수 통제 | 오라, 광역기, 거대 실루엣으로 전장 스케일 상승 |
| 보스전 | 긴장, 절정 | 보스 패턴 대응, 핵심 타워 보호 | 카메라/조명/이펙트가 한 판의 클라이맥스를 형성 |

## 4. MVP 범위 제안

### 절대 처음부터 다 만들지 말아야 하는 이유

이 프로젝트는 `3D`, `캐릭터 조작`, `타워디펜스`, `랜덤 드로우`, `합성`, `속성별 개성`, `고퀄 연출`이 동시에 들어간다. 이 중 어느 하나도 가볍지 않다. 처음부터 맵 여러 개, 속성 7개, 적 수십 종, 메타 성장까지 넣으면 거의 확실하게 일정이 터진다.

가장 작은 재미있는 버전은 `랜덤 소환 -> 3합성 -> 웨이브 방어 -> 강한 시각적 승격감`이 성립하는 버전이다.

### MVP에서 꼭 필요한 것

- 1개 맵
- 1개 플레이어 캐릭터
- 고정 설치 노드 방식
- 속성 3개
- 역할 기반 타워 3종 + 대표 조합 3개
- 타워 1~5단계
- 적 4종 + 엘리트 1종 + 보스 1종
- 웨이브 8개 전후
- 랜덤 소환과 3합성
- 기본 보정 시스템
- 합성/5단계 전용 연출
- 최소한의 튜토리얼성 UX

### MVP에서 과감히 빼도 되는 것

- 자유 배치형 타워 설치
- 속성 5개 이상
- 맵 2개 이상
- 플레이어 액티브 스킬 다수
- 장비/스킬트리/영구 성장 메타
- 멀티플레이
- 타워 개별 특성 롤 옵션
- 고급 상태이상 상호작용
- 복수 보스 패턴 세트
- 상점, 유물, 정교한 덱빌딩 시스템

### 추천 MVP 속성 수

- `권장: 3개`
- `확장 여지 포함 최대: 4개`

추천 순서는 아래와 같다.

1. `불`
2. `물`
3. `땅`
4. `암흑`은 확장 1순위

이유는 세 가지다.

- 시각 식별성이 아주 강하다.
- 전투 역할이 직관적으로 나뉜다.
- VFX와 실루엣 차이를 적은 리소스로도 크게 만들 수 있다.

### 추천 MVP 타워 종류 수

`역할 3종 + 속성 3개` 구조를 권장한다. 다만 MVP에서 플레이어에게 먼저 보여줄 콘텐츠는 `대표 조합 3개`만 둔다.

권장 역할 코어는 아래와 같다.

- `Attack Tower`: 지속 화력 중심
- `Control Tower`: 둔화/라인 제어 중심
- `Impact Tower`: 무거운 일격/넉백 중심

MVP 대표 조합은 아래처럼 잡는다.

- `Attack + Fire`
- `Control + Water`
- `Impact + Earth`

이렇게 하면 시스템 구조는 `역할 기반`으로 단순하게 유지하면서도, 플레이어 체감은 불/물/땅 대표 타워처럼 선명하게 만들 수 있다. 이후 확장 단계에서 같은 역할에 다른 속성을 붙여 조합 수를 늘리면 된다.

### 추천 MVP 웨이브 구성

- 총 `8웨이브`
- `1~2웨이브`: 일반 적 위주
- `3웨이브`: 빠른 적 추가
- `4웨이브`: 탱커 추가
- `5웨이브`: 원거리/방해형 적 추가
- `6웨이브`: 혼합 웨이브
- `7웨이브`: 엘리트 웨이브
- `8웨이브`: 보스 웨이브

### 왜 이렇게 줄여야 하는가

- 플레이어가 실제로 재미를 느끼는지는 `규칙의 수`보다 `한두 개의 루프가 얼마나 선명한가`로 결정된다.
- 이 게임의 첫 성공 조건은 콘텐츠 양이 아니라 `합성했을 때 진짜 강해진 것처럼 보이는가`다.
- 3D에서 고퀄처럼 보이려면 `적은 종류를 강하게 연출`하는 쪽이 훨씬 유리하다.

## 5. 시스템 설계

### 5.1 캐릭터 이동/조작 구조

#### 목표

캐릭터 조작은 액션 게임처럼 복잡할 필요가 없다. 이 게임에서 캐릭터의 본질은 `전장 관리자`다. 따라서 조작은 직관적이고 빠르되, 타워 관리에 집중할 수 있어야 한다.

#### 권장 조작

- `WASD`: 이동
- `마우스`: 카메라 회전 또는 목표 지정
- `E/F`: 근처 노드 상호작용
- `Q`: 현재 바라보는 빈 노드에 랜덤 소환
- `R`: 근처 합성 가능 타워 합성
- `Shift`: 짧은 대시
- `Tab`: 전장 요약 오버레이

#### 설계 원칙

- 캐릭터 전투는 `보조적`이어야 한다.
- 기본 공격은 가능하되 화력의 중심은 타워여야 한다.
- 상호작용 범위는 넉넉하게 줘서 조작 스트레스를 줄인다.
- 플레이어는 설치 지점 사이를 오가며 "판단을 실행하는 존재"로 느껴져야 한다.

#### MVP 추천

- 기본 공격 1종
- 짧은 대시 1종
- 설치/합성/판매 상호작용
- 별도 액티브 스킬은 미포함

### 5.2 타워 설치 구조

#### 권장 방식: 고정 설치 노드

자유 배치형은 멋있어 보이지만, MVP에선 아래 문제가 크다.

- 경로 악용 가능성
- 충돌/가시성 문제
- UI/UX 복잡도 증가
- AI가 구현할 때도 예외가 급증

따라서 첫 버전은 `맵에 배치된 설치 노드(Tower Node)`에만 설치하는 방식을 추천한다.

#### 노드 구조

- 각 노드는 `빈 상태`, `타워 설치 상태`, `합성 대상 상태`, `비활성 상태`를 가진다.
- 노드는 길목, 코너, 종착점 근처 등 전략적 위치에 배치한다.
- 노드 수는 MVP 기준 `12~16개` 정도가 적절하다.

### 5.3 랜덤 타워 소환 규칙

#### 기본 규칙

- 빈 노드에서만 소환 가능
- 소환 비용은 골드 사용
- 소환 결과는 `활성화된 속성 풀 중 하나의 1단계 타워`
- 기본 확률은 속성별 동일 가중치

#### 추천 초기 흐름

- 초반 4회 소환은 완전 난수 대신 `초기 안정화 규칙`을 적용한다.
- 목적은 두 가지다.
  - 최소한 한 가지 합성 축이 생기게 하기
  - 속성 식별을 자연스럽게 학습시키기

#### 추천 보정 규칙

1. 첫 4회 소환 안에는 `최소 1쌍의 같은 속성`이 나오도록 한다.
2. 보드 점유율이 70% 이상인데 합성 가능 조합이 하나도 없으면, 다음 소환은 `현재 보드와 시너지가 나는 결과`의 가중치를 높인다.
3. 5회 연속으로 신규 합성 가능성이 생기지 않으면, 다음 소환은 `보정 소환(Guided Summon)`으로 바뀌어 2개 후보 중 1개를 고르게 한다.

이 정도면 랜덤의 긴장감은 유지하면서도 "아무 것도 안 되는 판"을 줄일 수 있다.

### 5.4 타워 합성 규칙 (3개 합성)

#### 기본 규칙

- 같은 `속성`, 같은 `단계`의 타워 3개가 있으면 합성 가능
- 합성은 자동이 아니라 `플레이어가 실행`
- 결과는 해당 속성의 `다음 단계 타워 1개`

#### 왜 자동 합성이 아니라 수동 합성인가

- 플레이어가 합성 타이밍을 전략적으로 결정할 수 있다.
- 타워 3개를 분산 유지할지, 상위 1개로 압축할지 판단이 생긴다.
- 합성 순간을 연출 이벤트로 만들기 쉽다.

#### 추천 구현 방식

- 합성 가능한 타워는 서로 `룬 라인`, `오라`, `아이콘`으로 연결 표시
- 플레이어가 그중 하나에 접근해 `R`을 누르면 합성
- 새 타워는 선택한 기준 노드에 생성

### 5.5 타워 단계 구조 (1~5단계)

| 단계 | 역할 | 전투 변화 | 시각 변화 |
|---|---|---|---|
| 1단계 | 기본 재료 | 단일 기본 공격 | 소형 정령/토템, 최소 VFX |
| 2단계 | 역할 명확화 | 상태이상 또는 보조 효과 강화 | 외형 부품 증가, 색 발광 강화 |
| 3단계 | 빌드의 중핵 | 광역 또는 특수 패턴 추가 | 실루엣이 크게 변함 |
| 4단계 | 전장 중심축 | 강력한 서브 스킬/오라/광역기 | 거대화, 바닥 문양, 지속 이펙트 |
| 5단계 | 런의 클라이맥스 | 궁극 패턴, 존재감 압도 | 영웅급 모델, 고유 등장 연출 |

#### 단계 상승 설계 원칙

- 단계가 오르면 `공격 방식`도 달라져야 한다.
- 단계가 오르면 `실루엣`이 멀리서도 달라 보여야 한다.
- 단계가 오르면 `대기 상태만 봐도 강해 보이는` 요소가 추가되어야 한다.

### 5.6 속성 시스템 구조

`AGENTS.md` 기준으로 타워의 본체는 `역할(Role)`이고, 속성(Element)은 그 위에 얹히는 `추가 효과 데이터 + 시각 패키지`다.

즉, 타워는 "무엇을 하는가"로 정의하고, 속성은 "어떻게 때리고 어떤 부가 효과를 내며 어떻게 보이는가"를 정의한다.

#### 권장 역할 코어

- `Attack`: 단일/연사 중심의 기본 화력
- `Control`: 둔화/속박/라인 안정화
- `Impact`: 넉백/충격파/장갑 파괴

#### MVP 대표 조합

- `Attack + 불`
- `Control + 물`
- `Impact + 땅`

#### MVP 속성 효과 구조

- `불`: 폭딜/폭발/화상
- `물`: 둔화/빙결/라인 제어
- `땅`: 넉백/충격파/장갑 파괴

#### 확장 속성 효과 구조

- `암흑`: 저주/연쇄 폭발/처형
- `빛`: 관통/보호/정화
- `숲`: 소환수/속박/재생 억제
- `금`: 레이저/반사/경제 보조

### 5.7 적 웨이브 구조

#### 기본 구조

- 적은 정해진 경로를 따라 코어로 이동
- 웨이브는 `적 타입 조합`, `수량`, `간격`, `엘리트 여부`로 구성
- 웨이브 시작 전 10~20초 준비 시간 제공

#### MVP 맵 구조 권장

- 단일 메인 루트
- 코너 2~3개
- 중간 병목 1개
- 보스전에서만 일부 패턴으로 루트 압박 추가

### 5.8 보스 구조

보스는 체력만 높은 적이 아니라 `현재 빌드의 약점을 드러내는 시험`이어야 한다.

#### 보스 설계 원칙

- 보스는 일반 웨이브와 다른 `명확한 패턴`을 가진다.
- 보스는 플레이어에게 `타워 위치`, `합성 우선순위`, `캐릭터 위치`를 다시 생각하게 해야 한다.
- 보스전에는 시각/음향/카메라의 연출 피크가 필요하다.

### 5.9 자원 구조

#### MVP 권장 자원

- `골드`: 소환과 일부 관리 행동에 사용
- `코어 체력`: 누수 허용치를 나타냄
- `보정 게이지(공명 또는 운명 게이지)`: 나쁜 운을 완화하는 시스템 자원

#### 골드 사용처

- 랜덤 소환
- 판매 후 재정비 유도
- 향후 확장 시 리롤/축복/임시 버프에 활용 가능

#### 보정 게이지 사용처

- 보정 소환 발동
- 다음 소환에서 2개 후보 중 선택
- 또는 현재 보드와 맞는 결과 가중치 증가

### 5.10 초반 운빨 망 방지용 보정 시스템

초반이 망하면 플레이어는 시스템을 배우기도 전에 게임을 포기한다. 따라서 `초반만큼은 의도된 난수 설계`가 필요하다.

#### 추천 구조: Draw Director

- 게임은 겉보기에는 랜덤으로 보이되 내부적으로는 보드를 검사한다.
- 검사 항목
  - 플레이어가 최소한의 DPS를 갖췄는가
  - 합성 가능한 2중/3중 후보가 있는가
  - CC가 전혀 없는가
  - 보드 공간이 막혔는가

#### 보정 우선순위

1. 아무 역할도 못 하는 보드를 방지
2. 너무 오랫동안 합성이 안 되는 상황 방지
3. 특정 속성만 지나치게 몰리는 상황 완화
4. 그래도 결과는 "완전 통제"가 아닌 "조금 도와주는 랜덤"으로 유지

### 5.11 난이도 상승 구조

난이도는 단순 체력/공격력 상승만으로는 지루하다. `플레이 패턴을 흔드는 방식`으로 올라가야 한다.

#### 난이도 축

- 적 체력 증가
- 이동 속도 증가
- 방어 타입 추가
- 특수 적 비중 증가
- 엘리트 적에 보조 능력 부여
- 보스 패턴 복합화

#### 체감 난이도 설계

- 초반은 빌드 학습
- 중반은 역할 분담 시험
- 후반은 합성 효율과 전장 압박 시험
- 보스전은 "현재 빌드의 구멍"을 찌르는 구조

## 6. 타워/속성 설계

### 기준 구조: 역할 코어 + 속성 오버레이

시스템 구조는 역할 기반으로 잡고, 플레이어 체감은 속성 중심으로 읽히게 만드는 것이 가장 현실적이다. 즉 `Attack/Control/Impact`가 뼈대고, `불/물/땅`이 시각 언어와 부가 효과를 입힌다.

MVP에서는 모든 역할-속성 조합을 다 만들지 않는다. 대신 아래 3개를 전면에 세워 "속성별 대표 타워"처럼 느끼게 한다.

- `Attack + Fire`
- `Control + Water`
- `Impact + Earth`

### 대표 조합 기준 속성 정체성 정의

MVP에서는 플레이어가 숫자를 읽지 않아도 "불은 터뜨린다", "물은 묶는다", "땅은 밀어낸다"를 바로 이해해야 한다. 아래 내용은 `역할 코어 위에 얹힌 대표 속성 조합` 기준으로 정리한다.

### MVP 기준 우선 추천 속성

| 우선도 | 속성 | 이유 |
|---|---|---|
| 1 | 불 | 딜과 폭발 연출이 직관적이고 시각 만족감이 높음 |
| 2 | 물 | 둔화/빙결이 게임 이해를 쉽게 만들고 색이 잘 읽힘 |
| 3 | 땅 | 묵직한 임팩트와 넉백으로 전투 리듬이 좋아짐 |
| 4 | 암흑 | 확장 시 가장 화려한 차별화가 가능 |

### 6.1 불 속성

#### 역할

- 메인 딜러
- 군집 처리
- 지속 화상과 폭발 마무리

#### 전투 스타일

- 기본은 고화력 탄환
- 중반부터 화상 누적
- 상위 단계에서 폭발 반경 증가
- 후반에는 적 사망 시 연쇄 폭발 유도

#### 시각 스타일

- 주색: 붉은색, 주황색, 황금빛
- 보조 요소: 재, 불티, 용암 균열, 열 왜곡
- 형태: 위로 치솟는 화염, 뾰족한 창, 날카로운 실루엣

#### 공격 연출 아이디어

- 1단계: 작은 화염 구체 발사
- 2단계: 폭발성 불꽃 탄
- 3단계: 화염 부채꼴 분사 또는 작은 메테오 낙하
- 4단계: 지면 균열과 함께 분화구형 폭발
- 5단계: 거대한 화조 또는 용암 심장 형태가 주기적으로 대폭발 유발

#### 1~5단계 시각 진화 방향

- 1단계: 횃불형 정령
- 2단계: 이중 화염 토템
- 3단계: 마그마 코어를 품은 수호상
- 4단계: 불꽃 갑주를 두른 인페르노 센티넬
- 5단계: 피닉스/화산 심장급 존재로 승격

### 6.2 물 속성

#### 역할

- 둔화
- 빙결
- 라인 안정화

#### 전투 스타일

- 기본은 연속 투사체
- 적에게 둔화 중첩
- 상위 단계에서 빙결, 튕김, 범위 파동 추가
- 후반에는 적을 일정 구간에 묶는 제어형 역할

#### 시각 스타일

- 주색: 청록, 푸른색, 옅은 백색
- 보조 요소: 물결 리본, 안개, 얼음 결정, 수면 반사
- 형태: 곡선형, 유선형, 부드러운 회전

#### 공격 연출 아이디어

- 1단계: 물 탄환
- 2단계: 얼음 파편 사출
- 3단계: 적 사이를 튕기는 냉기 구체
- 4단계: 원형 파동으로 대규모 둔화
- 5단계: 거대한 해룡/빙하 문양이 나타나 구역을 얼림

#### 1~5단계 시각 진화 방향

- 1단계: 작은 수령 또는 물방울 토템
- 2단계: 얼음 결정을 품은 분수형 구조
- 3단계: 뱀처럼 말린 수룡 기둥
- 4단계: 파도 성소 또는 빙정 수호탑
- 5단계: 레비아탄/빙하 신전급 존재

### 6.3 땅 속성

#### 역할

- 병목 강화
- 넉백
- 장갑 파괴 또는 단단한 적 대응

#### 전투 스타일

- 공격 속도는 느리지만 한 방이 무겁다
- 상위 단계에서 지면 충격파와 에어본/넉백 발생
- 후반에는 방어력 감소, 광역 균열, 진동장으로 라인 고정

#### 시각 스타일

- 주색: 갈색, 황토, 금빛 석재, 짙은 녹회색
- 보조 요소: 바위 파편, 먼지, 균열, 부유 암석
- 형태: 육중한 하단, 낮고 넓은 무게 중심

#### 공격 연출 아이디어

- 1단계: 돌 탄환 발사
- 2단계: 작은 충격파
- 3단계: 지면에서 솟는 암석 첨탑
- 4단계: 전방 넉백 파열
- 5단계: 대지의 심장이 뛰듯 광역 지반 파괴

#### 1~5단계 시각 진화 방향

- 1단계: 돌무더기/카른
- 2단계: 룬이 새겨진 석주
- 3단계: 반쯤 깨어난 골렘형 포탑
- 4단계: 거석 수호자
- 5단계: 타이탄/모놀리스급 존재

### 6.4 암흑 속성 (확장 1순위)

#### 역할

- 저주 중첩
- 처형
- 연쇄 반응

#### 전투 스타일

- 기본은 약하지만 표식을 쌓을수록 강해짐
- 특정 체력 이하 적 처형
- 상위 단계에서 사망 연쇄 폭발

#### 시각 스타일

- 주색: 보라, 검정, 심연 청색
- 보조 요소: 그림자 리본, 흡수, 일식 고리, 왜곡
- 형태: 비대칭, 갈고리형, 공허 공간감

### 합성 후 시각적으로 반드시 달라져야 하는 점

합성은 단순히 크기만 키우면 안 된다. 아래 항목 중 최소 4개 이상이 바뀌어야 한다.

1. `실루엣`: 멀리서 봐도 다른 유닛처럼 보여야 함
2. `대기 연출`: 떠다니는 파편, 오라, 룬, 주변 입자
3. `공격 준비`: 충전 시간, 발광, 바닥 문양
4. `투사체`: 크기, 궤적, 잔상
5. `피격 효과`: 폭발 크기, 충격파, 잔류 장판
6. `사운드`: 레이어 추가, 저역 강화, 위압감
7. `바닥 존재감`: 그림자, 문양, 라이트 풀

### 5단계 타워가 특별해야 하는 이유

5단계는 희귀해서 특별한 것이 아니라, `이 게임의 장르 정체성을 완성하는 보상`이라서 특별해야 한다.

- 랜덤 소환의 긴장이 5단계를 향한 기대감으로 누적된다.
- 합성 시스템의 궁극 보상으로 기능한다.
- 플레이어가 "이번 판의 주인공"을 만들었다는 감정을 준다.
- 스트리밍/영상/스크린샷에서 가장 강한 홍보 포인트가 된다.

따라서 5단계는 반드시 아래를 만족해야 한다.

- 첫 등장 연출이 있다.
- 공격 패턴이 명확히 새롭다.
- 전장 조명과 주변 반응이 있다.
- 같은 속성 하위 단계와 `계보는 이어지되 다른 존재처럼` 보여야 한다.

## 7. 적/보스 설계

### 일반 적 타입 구분

| 타입 | 특징 | 플레이 영향 |
|---|---|---|
| 기본 보병 | 평균 체력/속도 | 빌드의 기본 화력 체크 |
| 빠른 적 | 낮은 체력, 높은 속도 | 둔화/집중 화력 필요 |
| 탱커 | 높은 체력, 느린 속도 | 폭딜/장갑 파괴 필요 |
| 원거리 방해형 | 타워 노드 또는 캐릭터를 방해 | 플레이어 이동 개입 유도 |
| 특수 적 | 분열, 보호막, 위상 이동 등 | 특정 속성의 역할 확인 |

### 타입별 전략 영향

#### 빠른 적

- 물 속성의 가치가 올라간다.
- 병목 구간 배치가 중요해진다.
- 플레이어가 누수 구간에 직접 달려가 보조해야 한다.

#### 탱커

- 불의 폭발 딜과 땅의 장갑 파괴 가치가 올라간다.
- 단일 타워 숫자보다 고단계 타워 효율이 중요해진다.

#### 원거리 방해형

- 타워만 세우고 방치하는 운영을 막는다.
- 플레이어 캐릭터 존재 이유를 만든다.

#### 특수 적

- 단순 DPS 체크를 넘어 `빌드 균형`을 요구한다.

### MVP 추천 적 구성

- 기본 보병 1종
- 빠른 적 1종
- 탱커 1종
- 원거리 방해형 1종
- 엘리트 1종
- 보스 1종

### 기믹형 보스 패턴 예시

#### 보스 예시 1: 지맥 파쇄자

- 거대한 암석 거수
- 특정 주기마다 전장을 내려쳐 `일부 노드를 잠시 비활성화`
- 체력이 일정 이하가 되면 병목 지점에 바위 장벽 생성
- 땅 속성만의 싸움이 아니라, `라인 유지와 노드 재가동 대응`이 핵심

#### 보스 예시 2: 심연의 행진자

- 전방 피해를 흡수하는 장막 전개
- 일정 시간마다 그림자 분신을 소환
- 저주 표식이 붙은 타워는 공격 속도가 느려짐
- 플레이어는 특정 코어 약점 노출 타이밍에 화력을 집중해야 함

#### 보스 예시 3: 서리 홍수의 여제

- 이동 경로 일부를 얼려 적 이동 속도를 변경
- 소환된 잡몹이 보스 보호막을 유지
- 보호막을 끊지 못하면 보스가 전장 전체에 빙결 파동 발동

### MVP 보스 추천

`지맥 파쇄자`를 추천한다.

이유는 아래와 같다.

- 땅/불/물 셋 모두 대응 역할을 만들기 좋다.
- 비주얼이 3D에서 묵직하게 잘 나온다.
- 노드 비활성화라는 기믹이 "캐릭터가 직접 뛰어다니는 이유"를 자연스럽게 만든다.

## 8. 그래픽/아트 디렉션

이 게임의 성패는 시스템 완성도만큼이나 `화면이 얼마나 맛있게 보이느냐`에 달려 있다. 여기서는 무조건 현실 그래픽을 목표로 하기보다, `적은 리소스로도 고퀄처럼 보이는 스타일화된 프리미엄 판타지`를 추천한다.

### 전체 비주얼 톤 제안

`스타일라이즈드 하이 판타지 디오라마`가 가장 적합하다.

핵심은 아래 세 가지다.

- 환경은 절제된 중립 톤
- 속성과 이펙트는 높은 채도와 강한 발광
- 타워와 보스는 실루엣과 VFX로 존재감을 만든다

즉, 배경은 조용하지만 전투 오브젝트는 과감해야 한다.

### 카메라 시점 추천

#### 권장 시점

- `세미 아이소메트릭 3D`
- 지면을 35~45도 정도 내려다보는 시점
- 카메라는 캐릭터를 따라가되, 전장 전체를 충분히 보여줌

#### 이유

- 캐릭터를 보여주면서도 디펜스 판 읽기가 가능하다.
- 타워의 실루엣이 잘 보인다.
- 5단계 등장 연출과 보스 연출에 카메라 줌/연출을 넣기 쉽다.

### 맵 분위기 추천

MVP는 `고대 엘리멘탈 성소` 콘셉트를 추천한다.

- 부서진 석재 플랫폼
- 금빛 룬이 흐르는 노드
- 맵 아래는 심연 또는 구름 바다
- 메인 컬러는 회색/황토/청회색

이 구조의 장점은 속성 색이 아주 잘 뜬다는 점이다.

### 색채 방향

환경은 색을 줄이고, 타워와 적 이펙트가 색을 가져가야 한다. 그래야 전투가 읽힌다.

#### 권장 컬러 밸런스

- 환경: `저채도 중립색`
- 아군 타워: `고채도 속성색`
- 적: `붉은 위협색 + 타입별 보조색`
- 보스: `중립 몸체 + 약점/패턴 강조색`

### 속성별 컬러 랭귀지

| 속성 | 메인 컬러 | 보조 컬러 | 감정 키워드 |
|---|---|---|---|
| 불 | `#FF6A1A` | `#FFD36A` | 뜨거움, 공격성, 폭발 |
| 물 | `#00B7FF` | `#BFEFFF` | 냉기, 흐름, 제어 |
| 땅 | `#8A6A43` | `#D8B982` | 무게, 안정감, 충격 |
| 암흑 | `#6A39FF` | `#140C20` | 저주, 심연, 위압 |

### 실루엣 설계 방향

실루엣은 멀리서도 읽혀야 한다. 색만으로 구분하면 전투가 복잡해질 때 금방 무너진다.

#### 속성별 실루엣 규칙

- 불: 위로 솟는 날카로운 피크, 불꽃 뿔, 가시
- 물: 곡선, 물방울, 소용돌이, 유선형
- 땅: 낮고 넓은 하중, 석주, 둔중한 사각형
- 암흑: 비대칭, 공허 공간, 초승달형 윤곽

### 단계 상승 시 비주얼 업그레이드 원칙

각 단계 상승 시 최소한 아래 변화가 있어야 한다.

1. 크기 증가
2. 실루엣 복잡도 증가
3. 발광 면적 증가
4. 바닥 문양 또는 오라 추가
5. 유휴 상태 애니메이션 추가
6. 공격 준비 동작 추가
7. 피격 연출 레이어 추가

#### 매우 중요한 원칙

`단계 상승 = 모델만 조금 바뀌는 것`이 아니라 `존재감의 계단`이 되어야 한다.

### 파티클/VFX 방향

#### 기본 원칙

- 한 번에 너무 많은 작은 파티클보다 `큰 읽힘`이 중요하다.
- 공격 준비, 발사, 명중, 잔류를 분리해서 설계한다.
- 속성별로 "한 방에 알아보는 모양"이 있어야 한다.

#### 속성별 VFX 핵심

- 불: 화염 혀, 폭발 링, 열 왜곡, 재 비산
- 물: 리본형 흐름, 안개, 얼음 파편, 구형 파동
- 땅: 먼지기둥, 돌 파편, 균열 데칼, 충격 링
- 암흑: 흡수 와류, 연기 띠, 일식 고리, 보라 잔상

### 라이팅 방향

#### 기본 조명

- 따뜻한 방향광 1개
- 차가운 환경광 1개
- 타워/보스에서 나오는 국소광은 제한적으로 사용

#### 조명 설계 목표

- 전장 전체는 읽히고
- 속성 공격은 더 빛나고
- 5단계와 보스는 조명만으로도 존재감이 생기게 한다

### 후처리(Post-Processing) 방향

- Bloom: 강하되 과하지 않게
- Color Grading: 영화적이되 정보 손실 없게
- SSAO/Contact Shadows: 지면 접지감 강화
- Vignette: 보스전/궁극 연출에서만 약하게
- Motion Blur: 매우 약하거나 비활성
- Depth of Field: 컷신성 순간에만 제한 사용

### UI까지 포함한 전체 미감 방향

UI는 모바일 랜덤디펜스처럼 가벼운 카지노 느낌이 아니라, `마법 공학 인터페이스`처럼 보여야 한다.

#### 추천 톤

- 석재/금속 프레임
- 룬 문양
- 속성 색 발광
- 둥근 카드보다 `의식진/장치 패널` 같은 구조

### "보는 맛"을 만들기 위한 연출 포인트

1. 타워 소환 시 노드 룬이 켜지고 짧은 소환 섬광 발생
2. 합성 가능 상태가 되면 해당 타워들 사이에 에너지 라인이 연결
3. 합성 순간 주변 입자를 끌어당긴 뒤 폭발적으로 방출
4. 4단계부터는 타워 주변에 환경 반응이 생김
5. 5단계 완성 시 카메라가 잠깐 당겨지고 주변 조명이 반응
6. 보스 입장 시 맵 전체의 라이트와 사운드가 바뀜
7. 웨이브 종료 시 전장이 잠깐 숨을 고르는 정적을 줌

### 합성 순간의 연출

합성은 이 게임의 핵심 보상이다. 따라서 연출 비중을 높게 둬야 한다.

#### 연출 시퀀스 권장

1. 세 타워에 동시 발광
2. 에너지 라인이 중앙으로 수렴
3. 아주 짧은 시간 정지 또는 슬로우
4. 상위 단계 실루엣이 역광으로 먼저 드러남
5. 충격파와 함께 완성
6. 첫 공격은 평소보다 더 과장된 버전으로 재생

### 5단계 완성 순간의 연출

5단계는 일반 합성과 분리된 전용 순간이어야 한다.

- 카메라 줌인 또는 살짝 회전
- 배경음 일시 필터링
- 주변 적에게 짧은 경직 또는 공포 반응
- 전장 바닥에 대형 룬 생성
- 화면 가장자리 색보정이 속성 톤으로 잠시 바뀜

### 저비용으로도 고퀄처럼 보이게 만드는 방법

1. 환경보다 `타워와 이펙트`에 리소스를 집중한다.
2. 단계별 완전 신규 모델 대신 `코어 실루엣 + 추가 파츠` 방식으로 확장한다.
3. 머티리얼의 `Emissive`, `Fresnel`, `Dissolve`를 적극 활용한다.
4. 적은 수의 강한 VFX를 반복적으로 재사용한다.
5. 그림자와 접지감, 바닥 데칼만 잘 잡아도 오브젝트가 고급스럽게 보인다.
6. 사운드 레이어를 잘 쌓으면 시각 품질 체감이 크게 오른다.
7. 카메라 연출은 비용 대비 효과가 크므로 적극 활용한다.

## 9. UI/UX 설계

### 화면에 반드시 보여야 하는 정보

- 코어 체력
- 현재 골드
- 웨이브 번호와 남은 적 수
- 다음 웨이브 시작까지 시간
- 현재 소환 비용
- 보정 게이지 상태
- 합성 가능 여부
- 플레이어 근처 노드 상호작용 안내

### 권장 화면 구조

#### 상단 좌측

- 코어 체력
- 골드
- 현재 웨이브

#### 상단 우측

- 다음 웨이브 정보
- 적 타입 아이콘 미리보기
- 게임 속도 조절

#### 하단 중앙

- 상호작용 프롬프트
- 소환/합성/판매 단축키 안내

#### 월드 스페이스 UI

- 각 타워 위에 속성 아이콘과 단계 표시
- 합성 가능 시 빛나는 링과 연결선
- 빈 노드는 은은한 룬 표시

### 캐릭터 조작과 타워 관리가 동시에 가능한 UI 구조

핵심은 `문맥 기반 인터랙션`이다.

- 빈 노드 근처에서는 `소환`
- 설치된 타워 근처에서는 `정보 확인/판매`
- 합성 가능 타워 근처에서는 `합성`

즉, 모든 버튼을 항상 노출하지 않고 `플레이어 위치와 대상 상태`에 따라 UI를 바꾸는 것이 좋다.

### 랜덤 소환 UI

MVP에선 UI를 과하게 복잡하게 만들 필요가 없다.

#### 기본 소환

- 빈 노드에 접근
- `Q`를 누르면 즉시 소환
- 짧게 결과 카드 또는 속성 문양 팝업

#### 보정 소환

- 보정 게이지 발동 시 2개 후보 카드 노출
- 플레이어가 빠르게 하나 선택

### 합성 가능 상태 표시

합성 가능 여부는 절대 숨기면 안 된다.

#### 표시 방식

- 같은 조건 타워 3개에 동일한 색의 오라 부여
- 세 타워 사이에 얇은 에너지 라인 연결
- 플레이어가 가까이 가면 `R 합성` 프롬프트 노출

### 속성 식별성

속성 식별은 `색 + 아이콘 + 실루엣 + 공격 모양`이 동시에 작동해야 한다.

색만으로 구분하면 VFX가 겹칠 때 혼란스러워진다.

### 웨이브 정보 표시

- 현재 웨이브 번호
- 남은 적 수
- 다음 웨이브 적 구성 미리보기
- 보스 웨이브는 별도 강조

### 초보자도 이해하기 쉬운 UX 방향

1. 첫 판에서는 초반 3웨이브 동안 툴팁을 강하게 제공
2. 타워 3개가 모이면 합성 튜토리얼을 즉시 표시
3. 속성 설명은 긴 텍스트보다 짧은 키워드로 표현
4. 합성 후 강해진 결과를 숫자보다 연출과 한 줄 설명으로 체감시킴
5. 정보량은 줄이고, 상호작용 순간에만 필요한 내용을 띄움

## 10. 기술 설계 방향

### Unity 기준 추천

`Unity 6 LTS + URP`를 권장한다.

### 왜 Unity가 적합한가

- C# 기반이라 AI가 구조화된 코드 생성과 리팩터링을 하기 좋다.
- ScriptableObject 기반 데이터 작업이 편하다.
- Cinemachine, Timeline, Addressables, Shader Graph 등 필요한 도구가 잘 갖춰져 있다.
- 3D 액션과 타워디펜스의 혼합 구조를 구현하기에 생태계가 충분히 넓다.
- 솔로 또는 소규모 개발에서 반복 제작 속도가 빠르다.

### 왜 HDRP보다 URP를 먼저 추천하는가

- MVP 단계에서 반복 속도가 더 빠르다.
- 에셋 호환성과 빌드/성능 관리가 쉽다.
- 스타일라이즈드 프리미엄 비주얼은 URP로도 충분히 낼 수 있다.

HDRP는 향후 `고사양 PC 전용 비주얼 데모`를 목표로 할 때 재검토하는 것이 현실적이다.

### 프로젝트 구조를 어떻게 나누면 AI가 작업하기 쉬운가

핵심은 `정적 데이터`, `런타임 로직`, `씬 오브젝트 표현`을 분리하는 것이다.

#### 권장 원칙

- ScriptableObject는 `정적 정의`만 담는다.
- 실제 전투 상태는 `런타임 클래스`에서 관리한다.
- MonoBehaviour는 `표현/브리지` 역할만 맡긴다.
- 시스템 경계를 명확히 나눠 AI가 작은 단위로 수정할 수 있게 한다.

### AI가 구현하기 쉬운 설계 원칙

1. `한 클래스 한 역할`을 지킨다.
2. `God Manager`를 만들지 않는다.
3. 데이터는 가능한 한 에디터에서 수정 가능하게 둔다.
4. 합성, 웨이브, 소환 규칙은 코드에 하드코딩하지 말고 정의 데이터로 분리한다.
5. 디버그용 시드와 전투 로그를 남겨 재현성을 확보한다.
6. 프레젠테이션 레이어와 게임 규칙 레이어를 분리해 AI가 안전하게 수정할 수 있게 한다.

### 향후 확장을 고려한 설계 원칙

- 속성 추가 시 기존 코어 로직을 건드리지 않고 `데이터 추가 + 전용 행동 추가`로 끝나야 한다.
- 신규 적 타입은 웨이브 정의만 바꾸어 쉽게 편성 가능해야 한다.
- 보스는 일반 적과 동일 기반을 공유하되 패턴 모듈만 다르게 둔다.
- UI는 노드 상호작용과 런 상태 UI를 분리해서 확장한다.

## 11. 개발 로드맵

### Phase 1: 프로토타입

#### 목표

이 게임의 핵심 루프가 실제로 재미있는지 확인한다.

#### 구현 항목

- 플레이어 이동
- 고정 노드 설치
- 랜덤 소환
- 같은 타워 3개 합성
- 1종 적 경로 이동
- 1종 속성 타워
- 3~4개 테스트 웨이브
- 임시 카메라/임시 VFX

#### 검증 기준

- 소환과 합성이 기술적으로 안정적으로 동작하는가
- 플레이어가 전장을 돌아다니는 행위가 번거롭지 않은가
- 타워 3개를 모아 합성하는 순간이 재밌는가

#### 다음 단계 조건

- 10분 플레이가 지루하지 않다
- 설치/소환/합성 UX가 이해 가능하다
- 최소 1개의 "강해졌다"는 감정이 발생한다

### Phase 2: MVP

#### 목표

한 판이 완결된 게임으로 성립하도록 만든다.

#### 구현 항목

- 속성 3개
- 타워 1~5단계
- 적 4종 + 엘리트 1종 + 보스 1종
- 웨이브 8개
- 보정 소환 시스템
- 코어 체력, 골드, 기본 UI
- 캐릭터 기본 공격과 대시
- 1개 맵 아트 1차 패스
- 합성 연출 1차 패스

#### 검증 기준

- 한 판 15~25분이 자연스럽다
- 랜덤이 억울하지 않다
- 불/물/땅이 서로 다르게 느껴진다
- 4단계 이상 타워에서 명확한 파워 스파이크가 보인다

#### 다음 단계 조건

- 5회 이상 반복 플레이 시 기본 루프가 유지된다
- 플레이어가 선호 속성을 말할 수 있을 정도로 정체성이 분명하다
- 보스전이 최소한 한 번은 기억에 남는다

### Phase 3: 비주얼 업그레이드

#### 목표

게임의 상업적 매력을 만드는 "보는 맛"을 크게 끌어올린다.

#### 구현 항목

- 타워 단계별 모델/머티리얼 정교화
- 속성별 고유 파티클 세트
- 합성 시네마틱 강화
- 5단계 전용 등장 연출
- 보스 연출 강화
- 라이팅/후처리 정리
- UI 아트 패스
- 사운드 디자인 강화

#### 검증 기준

- 스크린샷만 봐도 속성과 단계가 읽힌다
- 합성 순간을 보고 다시 만들고 싶어진다
- 5단계가 영상 클립으로 따로 잘라낼 만한 임팩트가 있다

#### 다음 단계 조건

- 플레이어가 기능보다 비주얼 칭찬을 먼저 한다
- 전투 가독성을 해치지 않으면서 화려함이 상승했다

### Phase 4: 콘텐츠 확장

#### 목표

장기 플레이 동기를 만드는 확장 콘텐츠를 올린다.

#### 구현 항목

- 암흑 등 신규 속성 추가
- 신규 맵
- 보스 추가
- 적 특수 패턴 다양화
- 메타 해금/업적/도전과제
- 난이도 모드
- 속성 간 조합 시너지 확장

#### 검증 기준

- 신규 콘텐츠가 기존 시스템 위에 자연스럽게 올라간다
- 밸런스 붕괴 없이 선택지가 늘어난다
- 반복 플레이 동기가 뚜렷해진다

#### 다음 단계 조건

- 새 속성/새 맵을 추가해도 코드 구조가 흔들리지 않는다
- 운영 지표 또는 플레이 테스트에서 반복 플레이 의사가 유지된다

## 12. 리스크 분석

### 12.1 이 프로젝트가 실패하기 쉬운 이유

이 프로젝트는 "겉보기에는 명확하지만 실제로는 매우 복합적인 게임"이다. 특히 아래 조합이 위험하다.

- 3D 제작 비용
- 캐릭터 조작
- 타워디펜스 밸런스
- 랜덤성
- 합성 구조
- 고퀄 비주얼 욕심

하나만 삐끗해도 전체 체감 품질이 크게 떨어질 수 있다.

### 12.2 범위 폭발 위험

#### 위험

- 속성을 너무 많이 넣고 싶어짐
- 각 속성마다 세부 타워를 여러 개 넣고 싶어짐
- 맵, 보스, 메타 시스템을 동시에 만들고 싶어짐

#### 방지책

- 첫 버전은 반드시 `1맵, 3속성, 1보스`로 고정
- 자유 배치 포기, 고정 노드 사용
- 메타 성장 시스템은 MVP 이후로 미룸
- "새 콘텐츠 추가" 전에 "합성 연출 품질"을 먼저 높임

### 12.3 랜덤성과 밸런싱 위험

#### 위험

- 운이 나쁘면 합성이 전혀 안 됨
- 운이 좋으면 게임이 너무 쉽게 무너짐
- 속성 간 효율 차이가 심하면 특정 속성만 쓰게 됨

#### 방지책

- 보정 소환 시스템 도입
- 전투 로그와 시드 재현 기능 확보
- 웨이브별 평균 화력 요구치를 수치화
- 각 속성이 최소 1개는 확실한 역할을 갖게 설계

### 12.4 그래픽 욕심 때문에 일정이 터질 위험

#### 위험

- 모델, 이펙트, 라이팅, 애니메이션을 모두 최고 퀄리티로 하려다 일정 붕괴

#### 방지책

- 환경보다 타워/보스/합성 연출에 집중
- 스타일라이즈드 방향으로 확정
- 단계별 모델을 완전 신규 제작보다 확장형 키트 방식으로 제작
- "한 속성 완성본"을 먼저 만들어 품질 기준을 고정

### 12.5 AI로 개발할 때 구조가 꼬일 위험

#### 위험

- AI가 임시 코드를 쌓아가며 구조가 오염됨
- 시스템 경계가 없으면 작은 수정이 전체를 흔듦
- ScriptableObject와 런타임 상태가 뒤섞일 수 있음

#### 방지책

- 모듈 경계 문서화
- 각 시스템마다 책임을 명확히 고정
- 기능 단위 브랜치/태스크로 쪼개 작업
- 리뷰 기준을 "동작"뿐 아니라 "구조 보존"까지 포함
- 프로토타입 코드와 MVP 코드를 동일선상에 두지 말고 정리 구간을 둠

### 12.6 플레이 감각이 산만해질 위험

#### 위험

- 캐릭터 조작과 타워 관리가 서로 방해할 수 있음
- 화면 이펙트가 화려한데 가독성이 떨어질 수 있음

#### 방지책

- 플레이어 액션 수를 최소화
- 상호작용은 문맥 기반으로 단순화
- 속성별 VFX 모양을 분명히 나눔
- 보스와 5단계만 강한 카메라 연출 사용

### 12.7 퍼포먼스 위험

#### 위험

- 다수 파티클, 실시간 라이트, 투사체가 겹치면 프레임 저하

#### 방지책

- 투사체와 이펙트 풀링
- 단계별 파티클 예산 설정
- 로컬 라이트 수 제한
- 상시 디버그 HUD로 프레임/VFX 수 체크

## 13. 최종 추천안

### 가장 현실적인 최종 개발안

첫 번째 목표는 "거대한 게임"이 아니라 `짧지만 한 판이 강렬한 3D 타워디펜스 런`을 완성하는 것이다. 이를 위해 가장 현실적인 방향은 아래와 같다.

- `PC 싱글플레이`
- `세미 아이소메트릭 3D`
- `고정 설치 노드`
- `속성 3개`
- `역할 3개 + 대표 조합 3개`
- `타워 5단계`
- `8웨이브 + 보스 1종`
- `강한 합성 연출`
- `보정 소환 시스템`

### 반드시 줄여야 할 것

- 자유 배치
- 속성 5개 이상
- 맵 다수
- 영구 성장 메타
- 플레이어 다중 스킬
- 세부 타워 분기 클래스 다수
- 과도한 적 특수 규칙

### 반드시 살려야 할 것

- 랜덤 소환의 기대감
- 같은 타워 3합성의 명확한 보상
- 속성별 전투/시각 정체성
- 캐릭터가 직접 전장을 관리하는 감각
- 5단계 타워의 압도적 연출
- 보스전의 기억 남는 한 장면

### 한 줄 결론

이 게임의 첫 성공 버전은 "많은 것을 넣은 게임"이 아니라, `세 가지 속성만으로도 매 판 강한 승격감과 보는 맛을 주는 게임`이어야 한다.

## 권장 개발 순서 요약

1. `고정 노드 + 랜덤 소환 + 3합성`만 있는 프로토타입을 만든다.
2. `한 속성`으로 먼저 재미와 가시성을 검증한다.
3. `불/물/땅` 3속성을 넣고 웨이브 8개 구조를 완성한다.
4. 보정 소환과 기본 UX를 넣어 억울한 판을 줄인다.
5. 보스 1종과 5단계 전용 연출을 붙여 런의 클라이맥스를 만든다.
6. 그 다음에야 모델, VFX, 라이팅, 후처리, UI 아트를 본격 강화한다.
7. MVP가 안정되면 `암흑`, 신규 맵, 메타 성장 같은 확장 요소를 올린다.
